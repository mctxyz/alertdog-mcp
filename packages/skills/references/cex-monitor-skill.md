# AlertDog CEX Monitor

## When To Use This Skill

Use this category when the task is about inspecting official CEX monitor feeds and signal data.

## Before You Start

- If the command needs authentication, run `alertdog auth show` first.
- If auth is missing, guide the user to run `alertdog auth set --apikey="$APIKEY"`.
- After auth is saved, use `alertdog run <tool-name>` and do not pass `--apikey`.
- Never ask the user to paste secrets into chat.
- Prefer loading only this file plus the root `SKILL.md` unless the user explicitly needs a broader catalog.

## `cex_price_subscription_create`

Use this tool only when the user explicitly wants to create a CEX price monitor subscription. This is an authenticated write operation with side effects. It creates a monitor using asset, monitor type, trigger type, volume filters, exchanges, notify interval, and notification channels.

### Command Summary

- Command: `alertdog run cex_price_subscription_create`
- Category: `cex-monitor`
- Auth: Uses saved local apiKey from `alertdog auth`
- Type: WRITE
- Risk: `caution`
- Safe to retry: no
- Recommended before: `cex_asset_search`, `notify_channel_list`

### Parameters

> `apiKey` is omitted from this table because `alertdog run` injects it from saved local auth.

| Parameter | Required | Type | Default | Rules | Description |
| --- | --- | --- | --- | --- | --- |
| `asset_id` | Yes | `integer` | `-` | min >= 0 | Asset id. Use 0 to monitor all assets. |
| `monitor_type` | Yes | `integer` | `-` | allowed: 1, 2 | Monitor type. 1 means percentage change, 2 means price. |
| `monitor_interval` | Yes | `integer` | `-` | allowed: 60, 300, 900, 1800, 3600, 21600, 43200, 86400 | Monitor time range in seconds. Supported values: 60, 300, 900, 1800, 3600, 21600, 43200, 86400. |
| `type` | Yes | `integer` | `-` | allowed: 1, 2, 3, 4, 5, 6 | Trigger type. 1 spot up, 2 spot down, 3 futures up, 4 futures down, 5 spot price, 6 futures price. |
| `vol_interval` | No | `string` | `-` | allowed: 5m, 10m, 15m, 30m, 1h, 4h, 6h, 12h, 1d | Volume candle interval. Leave it empty when volume filtering is not used. Supported values: 5m, 10m, 15m, 30m, 1h, 4h, 6h, 12h, 1d. |
| `min_quote_volume` | No | `string` | `-` | minLength 1 | Minimum quote volume threshold as a numeric string. Optional unless volume filtering is enabled. |
| `vol_volume_multiple` | No | `string` | `-` | minLength 1 | Minimum volume multiple threshold as a numeric string. Optional unless volume filtering is enabled. |
| `symbol` | Yes | `integer` | `-` | allowed: 1, 2 | Comparison operator. 1 means >=, 2 means <=. |
| `value` | Yes | `string` | `-` | minLength 1 | Threshold value as a numeric string. It can represent a percentage or a price. |
| `exchanges` | Yes | `array<string>` | `-` | minItems 1 | Exchange ids to monitor. Supported values: binance, okex, bybit, bitget, gate. |
| `notify_interval` | Yes | `integer` | `-` | allowed: 60, 180, 300, 600, 1800, 3600, 7200, 10800, 14400, 21600, 28800, 36000, 43200, 86400 | Notification interval in seconds. |
| `notify_history_retention` | No | `integer` | `-` | allowed: 0, 1 | Whether to retain notify history. Defaults to 1. |
| `channels` | Yes | `array<integer>` | `-` | minItems 1 | Notification channel ids. At least one channel id is required. |

### Conditional Rules

- When `channels`: channels must contain at least one notification channel id.
- When `exchanges`: Supported exchanges are binance, okex, bybit, bitget, gate.
- When `notify_interval`: notify_interval must use one of the supported second values shared by the CEX monitor tools.
- When `min_quote_volume or vol_volume_multiple is provided`: vol_interval is required only when volume filtering is enabled through min_quote_volume or vol_volume_multiple. support   5m 10m 15m 30m 1h 4h 6h 12h 1d

## Field Guidance

- `monitor_type`: Monitoring mode. Use 1 for percentage change monitoring and 2 for absolute price monitoring.
- `value`: Threshold value. For monitor_type=1 this is a percentage such as "0.01"; for monitor_type=2 this is the target price.
- `monitor_interval`: Monitoring time window in seconds. Supported values: 60, 300, 900, 1800, 3600, 21600, 43200, 86400.
- `type`: Trigger type. 1=spot rises above threshold, 2=spot falls below threshold, 3=futures rises above threshold, 4=futures falls below threshold, 5=spot price threshold, 6=futures price threshold.
- `vol_interval`: Volume candle interval. Supported values: 5m, 10m, 15m, 30m, 1h, 4h, 6h, 12h, 1d. This field is required only when volume filtering is enabled.
- `min_quote_volume`: Minimum quote volume threshold as a numeric string, such as "100000". Use it to enable minimum traded volume filtering.
- `vol_volume_multiple`: Minimum volume multiple threshold as a numeric string, such as "3". This compares current quote volume against historical average volume.
- `symbol`: Comparison operator. Use 1 for >= and 2 for <=.

### Example 1: Create a price monitor for Bitcoin

- CLI template: `alertdog run cex_price_subscription_create --asset_id <asset_id> --monitor_type <monitor_type> --monitor_interval <monitor_interval> --type <type> [--vol_interval <vol_interval>] [--min_quote_volume <min_quote_volume>] [--vol_volume_multiple <vol_volume_multiple>] --symbol <symbol> --value <value> --exchanges <exchanges> --notify_interval <notify_interval> [--notify_history_retention <notify_history_retention>] --channels <channels>`
- Example CLI: `alertdog run cex_price_subscription_create --asset_id 2 --monitor_type 1 --monitor_interval 60 --type 1 --symbol 1 --value 10 --exchanges okex,binance --notify_interval 300 --last_notify 0 --notify_history_retention 1 --channels 11331 --disabled 0`
### Example 2: Create a price monitor with volume filtering enabled

- CLI template: `alertdog run cex_price_subscription_create --asset_id <asset_id> --monitor_type <monitor_type> --monitor_interval <monitor_interval> --type <type> [--vol_interval <vol_interval>] [--min_quote_volume <min_quote_volume>] [--vol_volume_multiple <vol_volume_multiple>] --symbol <symbol> --value <value> --exchanges <exchanges> --notify_interval <notify_interval> [--notify_history_retention <notify_history_retention>] --channels <channels>`
- Example CLI: `alertdog run cex_price_subscription_create --asset_id 2 --monitor_type 1 --monitor_interval 60 --type 1 --vol_interval "" --min_quote_volume "" --vol_volume_multiple "" --symbol 1 --value 10 --exchanges okex,binance --notify_interval 300 --last_notify 0 --notify_history_retention 1 --channels 11331 --disabled 0`

## `cex_offical_candle_signal_feed_list`

Use this tool when the user wants to inspect the latest official CEX candle signal feed entries. This is an authenticated read-only operation that returns normalized pagination metadata and a list of current candle signal feed items.

### Command Summary

- Command: `alertdog run cex_offical_candle_signal_feed_list`
- Category: `cex-monitor`
- Auth: Uses saved local apiKey from `alertdog auth`
- Type: READ
- Risk: `safe`
- Safe to retry: yes

### Parameters

> `apiKey` is omitted from this table because `alertdog run` injects it from saved local auth.

| Parameter | Required | Type | Default | Rules | Description |
| --- | --- | --- | --- | --- | --- |
| `next` | No | `integer` | `-` | - | Pagination cursor. Use -1 for the first page, or pass the previous response page.next value to continue. |
| `limit` | No | `integer` | `-` | min > 0 | Maximum number of feed items to return. Defaults to 10. |
| `desc` | No | `boolean` | `-` | - | Whether to sort in descending order. Defaults to true. |

### Conditional Rules

- None.

## Response Field Guidance

- `id`: Official candle signal record id.
- `asset_id`: Internal asset id used by AlertDog for the detected coin or token.
- `asset`: Asset metadata for the signal target, including symbol, slug, display name, and icon.
- `cex_coin_price_id`: Spot market price record id associated with this signal snapshot.
- `exchange`: Exchange where the candle signal was detected, such as bitget or binance.
- `base_symbol`: Base asset symbol used by the exchange pair, such as ARIA.
- `raw_symbol`: Raw exchange trading pair symbol, usually a USDT pair such as ARIAUSDT.
- `interval`: Candle interval that triggered the signal, such as 5m, 15m, or 1h.
- `candle_start_at`: Start time of the candle window that produced the signal.
- `candle_end_at`: End time of the candle window that produced the signal.
- `price`: Observed signal price inside the detected candle window.
- `change_pct`: Percentage price change for the detected candle window.
- `avg_quote_volume`: Historical average quote volume used as the baseline for comparison.
- `volume_multiple`: Current quote volume divided by historical average quote volume.
- `quote_volume`: Current quote volume observed in the signal candle window.
- `createdAt`: Record creation time in AlertDog.
- `updatedAt`: Record update time in AlertDog.
- `spots`: Spot market snapshots for the same asset across supported exchanges, filtered to USDT quote pairs.
- `futures`: Futures market snapshots for the same asset across supported exchanges, filtered to USDT quote pairs.
- `spots[].cexCoinSpotPrice.marketPrice`: Latest spot market price for that exchange and USDT pair.
- `spots[].cexCoinSpotPrice.change1m~change1d`: Spot price change percentages across rolling windows from 1 minute up to 1 day.
- `futures[].cexFutureCoinPrice.markPrice`: Latest futures mark price for that exchange and contract pair.
- `futures[].cexFutureCoinPrice.fundingRate`: Current funding rate for the futures contract snapshot.
- `futures[].cexFutureCoinPrice.change1m~change1d`: Futures price change percentages across rolling windows from 1 minute up to 1 day.

### Example 1: Read the first page of official candle signal feed items

- CLI template: `alertdog run cex_offical_candle_signal_feed_list [--next <next>] [--limit <limit>] [--desc <desc>]`
- Example CLI: `alertdog run cex_offical_candle_signal_feed_list`
### Example 2: Read the next page with a returned cursor

- CLI template: `alertdog run cex_offical_candle_signal_feed_list [--next <next>] [--limit <limit>] [--desc <desc>]`
- Example CLI: `alertdog run cex_offical_candle_signal_feed_list --next 7724 --limit 10 --desc true`

## `cex_candle_signal_subscription_notify_history_list`

Use this tool when the user wants to inspect notification trigger history generated by the user's own CEX candle signal subscriptions. This is an authenticated read-only operation that returns normalized personal subscription history items together with prev and next cursors. When subscribe_id is omitted, it returns notification history across all subscriptions owned by the current user.

### Command Summary

- Command: `alertdog run cex_candle_signal_subscription_notify_history_list`
- Category: `cex-monitor`

- Type: READ

- Recommended before: `cex_candle_signal_subscription_list`

### Parameters

| Parameter | Required | Type | Default | Rules | Description |

| `cursor` | No | `integer` | `-` | min >= 0 | User subscription history cursor. Defaults to 0 for the latest page. |
| `limit` | No | `integer` | `-` | min > 0 | Maximum number of history records to return. Defaults to 100. |
| `desc` | No | `boolean` | `-` | - | Whether to sort in descending order. Defaults to true. |

### Conditional Rules

- When `subscribe_id is provided`: Use subscribe_id when you want to read trigger history for one specific user subscription only.

- When `pageFlip=prev`: Use pageFlip=prev when you want real-time monitoring or incremental updates. Reuse the previous response page.prev as cursor. If no new data is returned, keep the old cursor value.
- When `pageFlip=next`: Use pageFlip=next when you want to fetch all history from newest to oldest. Reuse the previous response page.next as cursor. If no additional history is returned, keep the old cursor value.

## Response Field Guidance

- `id`: Official candle signal record id.
- `asset_id`: Internal asset id used by AlertDog for the detected coin or token.
- `asset`: Asset metadata for the signal target, including symbol, slug, display name, and icon.
- `cex_coin_price_id`: Spot market price record id associated with this signal snapshot.
- `exchange`: Exchange where the candle signal was detected, such as bitget or binance.
- `base_symbol`: Base asset symbol used by the exchange pair, such as ARIA.
- `raw_symbol`: Raw exchange trading pair symbol, usually a USDT pair such as ARIAUSDT.
- `interval`: Candle interval that triggered the signal, such as 5m, 15m, or 1h.
- `candle_start_at`: Start time of the candle window that produced the signal.
- `candle_end_at`: End time of the candle window that produced the signal.
- `price`: Observed signal price inside the detected candle window.
- `change_pct`: Percentage price change for the detected candle window.
- `avg_quote_volume`: Historical average quote volume used as the baseline for comparison.
- `volume_multiple`: Current quote volume divided by historical average quote volume.
- `quote_volume`: Current quote volume observed in the signal candle window.
- `createdAt`: Record creation time in AlertDog.
- `updatedAt`: Record update time in AlertDog.
- `spots`: Spot market snapshots for the same asset across supported exchanges, filtered to USDT quote pairs.
- `futures`: Futures market snapshots for the same asset across supported exchanges, filtered to USDT quote pairs.
- `spots[].cexCoinSpotPrice.marketPrice`: Latest spot market price for that exchange and USDT pair.
- `spots[].cexCoinSpotPrice.change1m~change1d`: Spot price change percentages across rolling windows from 1 minute up to 1 day.
- `futures[].cexFutureCoinPrice.markPrice`: Latest futures mark price for that exchange and contract pair.
- `futures[].cexFutureCoinPrice.fundingRate`: Current funding rate for the futures contract snapshot.
- `futures[].cexFutureCoinPrice.change1m~change1d`: Futures price change percentages across rolling windows from 1 minute up to 1 day.
- `subscribe_id`: User subscription id that generated this trigger history record.
- `user_id`: User id that owns the candle signal subscription and its trigger history.
- `expire_at`: Expiration time for this history snapshot or retention window.
- `page.pageFlip`: Optional paging direction. Omit it for the default latest query. Use prev together with cursor=page.prev to move upward and fetch newer/latest data, or use next together with cursor=page.next to move downward and fetch older history.
- `page.prev`: Cursor to continue upward pagination for newer/latest data. For real-time updates, send this value back as cursor with pageFlip=prev. If the next response has no new data, keep using the previous cursor value instead of replacing it.
- `page.next`: Cursor to continue downward pagination for older history records. Send this value back as cursor with pageFlip=next. If the next response has no additional history, keep using the previous cursor value instead of replacing it.
- `page.latestCursor`: Convenience alias of page.prev. For incremental updates, send this value back as cursor with pageFlip=prev, and do not update the cursor when no new data is returned.
- `page.olderCursor`: Convenience alias of page.next for fetching older history records. Do not update the cursor when no additional history is returned.
